## 📖 Overview

Your task is to design and develop a **responsive personal or professional webpage** for a real person using **ONLY HTML and CSS**.

This assessment introduces a simplified **real-world design process** by encouraging you to interact with an actual person, understand their needs, and translate those needs into a functional, beautiful webpage.

## 🎯 Your Mission

You are required to:

2. 🧠 **Understand** what they want to communicate online

4. 💻 **Develop** the webpage using HTML and CSS

## 🧠 Mini Design Thinking Process

Before writing any code, follow this simple process:

### 1️⃣ Understand the Person
Ask your user simple questions such as:
* What do you do?
* Who is your target audience?
* What information is most important to display?
* What colors or visual style do you prefer?
* What action should visitors take? (e.g., call you, view a gallery, etc.)

### 2️⃣ Plan the Structure
Create a simple **hand-drawn sketch** of your webpage *(REQUIRED)*.
Your sketch should outline the Header, Navigation, Main sections, Footer, and Content organization to demonstrate planning before implementation.

### 3️⃣ Build the Webpage
Develop the webpage using ONLY HTML and CSS.

## 🛠️ Technical Requirements

### 🧱 HTML Requirements
- [ ] Semantic HTML tags (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`)
- [ ] Proper heading hierarchy (`<h1>` to `<h6>`)
- [ ] Internal navigation links
- [ ] Images with meaningful `alt` text

### 🎨 CSS Requirements
- [ ] External stylesheet usage
- [ ] Flexbox
- [ ] CSS Grid *(must be used at least once)*
- [ ] Media queries for responsive design
- [ ] Relative units (`rem`, `%`, `vh`, etc.)
- [ ] Typography styling, margins, padding, borders, and border-radius
- [ ] Colors and background colors
- [ ] Hover effects and/or transitions

> [!CAUTION]
> ### 🚫 Forbidden Technologies
> * ❌ Bootstrap, Tailwind CSS, or any CSS libraries
> * ❌ JavaScript / JS Frameworks
> * ❌ Inline styling
> 
> *You are expected to demonstrate your **own** HTML and CSS skills.*

## ✏️ Submission Details

### The Sketch (`sketch.jpg`)

Your sketch does NOT need to be artistic. The goal is planning and structure. It must show a rough layout idea, section arrangement, and navigation placement.

* **Allowed:** Pen/pencil, paper, phone photo.
* **Not Allowed:** Figma, Canva, Adobe XD, AI-generated wireframes, or digital mockups.

### The Banner Image (`banner.jpg`)

A real-world photo representing the person, business, or subject matter of the website. This image should be a genuine photo you took or obtained with 
permission, and it should not be AI-generated.

This is the bare minimum requirement. You can add additional images to the `images` folder to enhance your webpage design, but the `banner.jpg` must be included as a key visual element on your webpage.

### The Conversation Summary (`conversation.txt`)

A short paragraph or bulleted list describing who the person is, what they do, the purpose of the webpage, and their preferences.
**Example:**

> *"I interviewed a local photographer who wanted a simple portfolio webpage. The main audience is potential clients. They preferred dark colors and large images. The website should highlight photography samples and contact information."*
